File distribution service status shows red, but the service is still running and serving

Description

Customer : National Semiconductor
Version : 5.0r1-6

Customer reported that HTTP file distribution status shows as RED, however service is running and serving files.

We have tried to reproduce the problem as we suspected that changing VIP of the HA pair caused the service status to change and stuck..But we could not reproduce the issue..

We requested backup from customer and as soon as we loaded the data on lab unit ,we see the issue. Service status shows "0% - An HTTP file distribution daemon failure has occurred" in GUI..

Another side node, we can force the service to be stopped when the service status show "red", that is another bug I think (hidden behind this issue)
